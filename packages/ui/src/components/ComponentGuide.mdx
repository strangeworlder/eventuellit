# Komponenttiopas (Component Guide)

This page groups every `@repo/ui` component by **use-case** rather than atomic level.
Use it as the entry point to decide which component to look up next.

---

## Pelilogiikka (Gameplay mechanics)

| Component | When to use |
|-----------|-------------|
| `StationConnections` | Game board / world map. Renders a compass-rose of connected stations around a central node with animated framer-motion transitions. The most complex component in the library. |
| `DicePoolAllocator` | Pre-roll phase: let the player distribute a fixed pool of d20s across named action axes. Has an optional attribute die toggle per axis. |
| `DicePoolTracker` | During / post-roll: display an existing dice pool and let the player toggle individual dice as spent. Groups dice by face count. |
| `DiceRoller` | Simple one-click dice roll widget. Use for quick rolls when full allocator is not needed. |
| `DiceIcon` | A single decorative or semantic die face (d4 / d6 / d8 / d10 / d12 / d20 / swirl). Use standalone inside game UI or as a visual accent. |
| `StatBlock` | Read-only stat display. Shows a label, numeric value, optional max, and an icon. Use for character sheet display. |
| `ActiveStatBlock` | Interactive stat display with +/âˆ’ controls. Use when the value can change during play (e.g. HP, Sisu). |
| `EnduranceBlock` | Composite Kesto (endurance) widget: two sub-attributes with passive dice on the left, an increment/decrement Kesto value on the right. |
| `AttributeCard` | Character attribute card with dice visualisation. Converts an allocation value into the tiered d4â€“d20 dice representation. |
| `AspectTag` | Inline harm or skill tag chip. Use `variant="harm"` for conditions/wounds (healable), `variant="skill"` for acquired skills (removable/custom). |
| `SkillTagList` | A managed list of `AspectTag` skill items with inline add / rename / remove. Use on character sheets. |
| `SkillMasonry` | A browsable, searchable, obscurable masonry grid of skill items. Use for the skill catalogue view. |
| `GameTerm` | Inline typographic emphasis for game terms (e.g. **Kesto**, **Sisu**). Not a chip â€” purely inline text. |

## Navigaatio ja rakenne (Navigation & structure)

| Component | When to use |
|-----------|-------------|
| `TopNav` / `TopNavList` / `TopNavLink` / `TopNavDropdown` | Primary horizontal **navigation** (router). Renders a semantic `<nav>`; links use `NavLink` with `aria-current="page"`. Pair `TopNavDropdown` with `TopNavList` for small viewports. |
| `Tabs` / `TabsList` / `TabsTrigger` / `TabsContent` | **In-page tab panels** (state-driven). Full ARIA tab pattern (arrow keys, `tablist` / `tab` / `tabpanel`). Not for changing routes â€” use `TopNavLink` for that. |
| `Sidebar` | Collapsible side navigation panel. Contains a `SidebarNav` of `Button` items. Uses `useSidebar()` hook for controlled expand/collapse. |
| `Drawer` | Slide-in side panel. Use for contextual detail panels, not for primary navigation. `useDrawer()` hook for control. |
| `Dialog` | Modal dialog with focus trap, title, description, footer actions. Use for confirmations and forms requiring focused interaction. |
| `ArticleProgressNavigator` | Vertical progress rail for long-form content. Shows section markers and scroll progress. Used in ruleset / episodes articles. |
| `Link` | Styled anchor / router link. Use instead of `<a>` for all internal navigation. |

| Component | When to use |

## Teematietoisuus (Theme awareness)

All components accept an optional `theme?: Theme` prop that scopes `data-theme` to the component root.
Available themes: `base` | `inverted` | `primary-light` | `primary-dark` | `secondary-light` | `secondary-dark` | `accent-light` | `accent-dark`.
